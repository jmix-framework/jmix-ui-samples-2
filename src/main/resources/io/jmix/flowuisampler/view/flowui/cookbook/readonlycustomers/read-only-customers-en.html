This example demonstrates usage of a database view in the list view.

The Customer entity is mapped to the SAMPLER_CUSTOMER table which stores information about customers. The ReadOnlyCustomer entity is mapped to the SAMPLER_CUSTOMER_VIEW database view defined as follows:

create view SAMPLER_CUSTOMER_VIEW as select ID, NAME || ' ' || LAST_NAME as FULL_NAME, AGE from SAMPLER_CUSTOMER

The read-only-customers.xml view defines data components and table to load and display data from the database view using the ReadOnlyCustomer entity. Obviously, this entity cannot be used to edit data. So the table actions have custom handlers in the ReadOnlyCustomersSample controller to edit data using the Customer entity.

The action handlers use the DialogWindows bean to open the Customer edit screen and reload data in the table after successful commit.
