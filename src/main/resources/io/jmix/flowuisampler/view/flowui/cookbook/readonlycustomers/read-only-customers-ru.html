Данный пример демонстрирует использование представления (view) базы данных в экране списка.

Сущность Customer отображена на таблицу SAMPLER_CUSTOMER, хранящую информацию о заказчиках. Сущность ReadOnlyCustomer отображена на представление БД SAMPLER_CUSTOMER_VIEW, объявленное следующим образом:

create view SAMPLER_CUSTOMER_VIEW as select ID, NAME || ' ' || LAST_NAME as FULL_NAME, AGE from SAMPLER_CUSTOMER

Экран read-only-customers.xml определяет компоненты данных и таблицу для загрузки и отображения данных из представления БД с помощью сущности ReadOnlyCustomer. Разумеется, данная сущность не может использоваться для изменения данных, поэтому в контроллере ReadOnlyCustomersSample для действий таблицы созданы специальные обработчики, чтобы редактировать данные с помощью сущности Customer.

Обработчики действий используют бин DialogWindows чтобы открывать экран редактирования сущности Customer и перезагружать данные в таблице после успешного коммита.
