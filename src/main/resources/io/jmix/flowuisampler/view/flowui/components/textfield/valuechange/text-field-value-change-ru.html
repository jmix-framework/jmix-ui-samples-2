Параметром ValueChangeEvent задаётся режим передачи изменений на сервер для вызова события на серверной стороне. В платформе реализовано 4 режима передачи:

  • EAGER - событие вызывается незамедлительно после каждого изменения текста, то есть после каждого нажатия клавиш. Запросы отправляются по отдельности и обрабатываются последовательно один за другим. Тем не менее, асинхронная передача событий изменения на сервер позволяет не блокировать дальнейший ввод текста.

  • LAZY (по умолчанию) - событие вызывается во время паузы в наборе текста. Продолжительность паузы можно задать с помощью метода setValueChangeTimeout(). Событие изменения текста обрабатывается принудительно перед возможным событием ComponentValueChangeEvent, даже если пользователь не выдержал паузу в наборе текста.

  • BLUR - событие вызывается, когда текстовое поле теряет фокус.

  • TIMEOUT - событие вызывается после периода ожидания. В случае ввода нескольких изменений за один период, на сервер отсылается событие со всеми изменениями, включая последнее. Продолжительность периода ожидания можно задать с помощью метода setValueChangeTimeout().

  • ON_CHANGE (по умолчанию) - событие вызывается, вызывается changeEvent клиентской части.
