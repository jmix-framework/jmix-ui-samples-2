Композитные компоненты создаются путем объединения существующих компонентов. Они инкапсулируют макет и поведение пользовательского интерфейса и позволяют повторно использовать одинаковые части пользовательского интерфейса в разных представлениях. Составные компоненты могут содержать компоненты, поддерживающие данные, и сами загружать данные или полагаться на контейнеры данных, предоставляемые хост-экраном.

В этом примере мы демонстрируем композитные компоненты для редактирования адресов AddressComponent. Компонент содержит несколько TypedTextField и EntityComboBox для отображения различных частей адреса. Этот компонент работает с встраиваемой сущностью Address.

Конструктор компонента принимает несколько инфраструктурных бинов и создает внутренние компоненты и контейнеры данных. Метод initContent() вызывается когда композитный компонент будет подключен к дереву компонентов. Он добавляет свои внутренние компоненты в родительский компонент, который определен с типом (Composite<FormLayout>). Композитный компонент предоставляет в своем API метод setDataContainer() который должен быть вызван после того, как компонент будет создан в экране. Метод привяжет внутренние компоненты к переданному контейнеру данных.

После написания класса композитного компоненты, он может быть создан в экранах программно. См. initAddress() метод в классе AddressUsage.

Чтобы иметь возможность декларативно использовать новый компонент в XML экрана, необходимо выполнить еще несколько шагов:

  1. Создать XSD-файл для компонента (см. address-component.xsd).
  2. Реализовать загрузчик из XML (см. AddressComponentLoader.java).
  3. Зарегистрировать компонент и его загрузчик в приложении (см. AddressComponentRegistration.java).

После выполнения этих шагов составной компонент можно будет использовать в XML-дескрипторах экранов. См. файл address-usage.xml.
