**Timer** это facet, (невизуальный компонент) позволяющий выполнять некоторый код контроллера фрагмента через
определенные
промежутки времени.
Срабатывание таймера происходит в потоке обработки событий пользовательского интерфейса, что позволяет обновлять
фрагмент
без каких-либо ограничений.
Таймер прекращает работу при открепления фрагмента, для которого он был создан.

Для описания таймера используется элемент `timer`. Элемент имеет следующие атрибуты:

- `delay` - обязательный атрибут, в нем задается интервал срабатывания таймера в миллисекундах;
- `autostart` необязательный атрибут, при установке которого в `true`, таймер стартует сразу после
  прикрепления фрагмента. По умолчанию `false`, что означает что для старта таймера необходимо вызвать его
  метод `start()`;
- `repeating` - необязательный атрибут, включает многократное срабатывание таймера. Если значение атрибута
  равно `true`, то таймер выполняется циклически, через равные промежутки времени, заданные в атрибуте `
  delay`.В противном случае таймер выполняется один раз через `delay`  миллисекунд после старта
  таймера.

Для выполнения некоторого кода по таймеру, создайте подписку на событие `TimerActionEvent` в контроллере
фрагмента.

См. также <a href="/ui-samples/sample/delayed-data-load">пример</a> того, как использовать таймер для задержки
ресурсоемкой операции загрузки данных до тех пор, пока пользователь не выберет окончательные условия.
