Собственные UI-компоненты можно создавать на базе сторонних JavaScript библиотек. Данный пример демонстрирует интеграцию виджета jQuery UI Slider и его использование в экране Jmix.

Файл Slider.js, расположенный в каталоге frontend/src/component/slider, содержит веб-компонент, оборачивающий виджет jQuery UI Slider. Его метод ready() используется для инициализации виджета.

Серверная часть кастомного компонента определена в классе Slider.java. Он содержит несколько аннотаций для интеграции с клиентской частью:

  • @Tag - задает имя кастомного HTML-элемента веб-компонента.
  • @NpmPackage импортирует NPM-модуль.
  • @JsModule - импортирует JavaScript-модуль.
  • @CssImport - импортирует стили.

После создания клиентской и серверной частей компонента, его можно использовать в экранах путем создания экземпляра Java-класса и добавления в родительский компонент.

Ниже описаны шаги для полной интеграции компонента в Jmix UI:

  1. Создать XSD для компонента (см. slider.xsd).
  2. Реализовать загрузчик компонента (см. SliderLoader.java).
  3. Зарегистрировать компонент и его загрузчик в приложении (см. SliderComponentRegistration.java).

После завершения этих шагов компонент можно использовать в XML-дескрипторах экранов. См. файл slider-usage.xml.

Подробная информация о создании кастомных компонентов приведена в документации Vaadin: Importing JavaScript & CSS, Integrating Web Components, Element API.
