This sample demonstrates possible approaches for displaying PDF files from the file storage using the IFrame component and REST API.

This method is convenient if the document is stored in an entity as a FileRef.
If your application is configured with a Generic REST API you can simply specify the request address as the source for the IFrame (see Files API :: Jmix Documentation).
Thus, this request will receive a file that will be displayed using standard browser tools in IFrame.

In this example, a custom request controller was configured. See CustomFileDownloadController.java.
In order for this type of requests to be processed correctly by the application, it is required to add a special security filter.
See CustomRestSecurityConfiguration.java.

This source code contains a directive that allows the use of resources if the requests is sent from demo.jmix.io.
More detailed information is available in the documentation for the directive: CSP: frame-ancestors - HTTP | MDN 6.
