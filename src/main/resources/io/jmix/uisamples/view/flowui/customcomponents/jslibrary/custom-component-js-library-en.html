You can create custom UI components based on third-party JavaScript libraries. This example demonstrates integration of the jQuery UI Slider widget and its usage in a Jmix view.

The Slider.js file located in the frontend/src/component/slider folder contains a web component wrapping the jQuery UI Slider widget. Its ready() method is used to initialize the widget.

The server-side part of the custom component is defined in Slider.java. It contains a number of annotations for integration with the client-side part:

  • @Tag - specifies the name of the custom HTML element of the web component.
  • @NpmPackage imports an NPM module.
  • @JsModule - imports a JavaScript module.
  • @CssImport - imports styles.

After creating the client-side and server-side parts of the component, it can be used in views by creating an instance of its Java class and adding it to a parent component.

The following steps completely integrate the component into Jmix UI:

  1. Create an XSD for the component (see slider.xsd).
  2. Implement the component loader (see SliderLoader.java).
  3. Register the component and its loader in the application (see SliderComponentRegistration.java).

Once these steps are completed, the custom component can be used in view XML descriptors. See the slider-usage.xml file.

See more information about creating custom components in Vaadin documentation: Importing JavaScript & CSS, Integrating Web Components, Element API.
