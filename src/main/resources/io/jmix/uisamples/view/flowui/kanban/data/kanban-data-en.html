The Kanban board component has the ability to bind data. To do this, specify the dataContainer attribute. You can filter and edit data in the container, all changes will be displayed in the component.

To set the mapping of the attributes of the displayed entity with the attributes of the Kanban task card, you need to define the propertiesMapping object.

The Kanban task card has the following set of attributes:

  • id (required)
  • status (required)
  • text (required)
  • username
  • userAvatar
  • color
  • dueDate
  • progress
  • tags
  • priority
  • swimlane

The rules for displaying matching elements to Kanban task cards can be controlled using the component properties. Required attributes have default values. If the propertiesMapping object is not specified, the entity will use the id, status, and text fields to determine the mapping.
