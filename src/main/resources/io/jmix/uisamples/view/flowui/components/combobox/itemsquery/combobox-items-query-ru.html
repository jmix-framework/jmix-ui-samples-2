Компонент ComboBox поддерживает обратный вызов для получения элементов из серверной части:

Вы можете определить запрос декларативно, используя элемент itemsQuery в XML.

Элемент itemsQuery для ComboBox имеет следующие атрибуты:

  • escapeValueForLike - включает поиск значений, содержащих специальные символы: %, \ и т. д. Значение по умолчанию — false.
  • searchStringFormat - строка Groovy. C её помощью, вы можете использовать любые допустимые строковые выражения Groovy.

Примечание: entityClass и fetchPlan отсутствуют, поскольку ComboBox предполагает, что будут загружены скалярные значения. В случае, если требуются, сущности используйте EntityComboBox.

Если itemsQuery не определен, вы можете программно установить список элементов, используя setItemsFetchCallback в ViewController.

Примечание: Поскольку реальная работа делегируется com.vaadin.flow.data.provider.HasLazyDataView#setItems(com.vaadin.flow.data.provider.CallbackDataProvider.FetchCallback, экземпляр QueryTrace передается обратному вызову fetch.

В результате - несколько ограничений:

  • Необходимо вызвать методы query.getOffset() и query.getLimit(), в противном случае выбрасывается исключение.
  • Количество элементов, возвращаемых обратным вызовом, не может превышать значение limit, в противном случае выбрасывается исключение.
