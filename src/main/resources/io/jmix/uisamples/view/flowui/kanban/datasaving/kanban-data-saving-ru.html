Существует два способа сохранения изменений для компонента доски Kanban: ручной (по-умолчанию) и автоматический.

С помощью компонента возможно изменять три атрибута сущности:

  • status, swimlane - с помощью перетаскивания карточек задач Kanban в различные столбцы
  • priority - с помощью соответствующего выпадающего списка рядом с текстом на карточке задачи Kanban

Изменения автоматически фиксируются в контейнере данных, однако по-умолчанию не инициируют сохранение в базу данных.

В ручном режиме необходимо определить SaveDelegate, который будет вызываться каждый раз при изменении карточек с помощью доски Kanban.

В автоматическом режиме небходимо установить атрибут autoSave в XML в значение true. Таким образом в компонент автоматически установится SaveDelegate, который будет сохранять DataContext текущего экрана.
Будьте аккуратны, весь контекст экрана будет сохранён.
