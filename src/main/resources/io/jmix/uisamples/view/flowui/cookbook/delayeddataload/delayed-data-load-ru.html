В этом примере показано, как можно отложить затратную операцию загрузки данных до тех пор, пока пользователь не сделает окончательный выбор условий.

Попробуйте выбрать несколько строк в левой таблице, отображающей заказы. Правая таблица будет загружена через 2 секунды после последнего выбора и покажет все элементы выбранных заказов с расчетом общего количества, сгруппированные по названию продукта.

В примере используется таймер, который перезапускается при каждом изменении выбора. Загрузка данных выполняется, когда срабатывает последний таймер, то есть через 2 секунды после последнего изменения выбора.

Левая таблица связана с обычным контейнером данных, содержащим коллекцию сущности Order. Загрузчик контейнера запускается в обработчике события BeforeShowEvent.

Правая таблица подключена к контейнеру key-value сущности, загруженному агрегированными данными. Его загрузчик запускается в обработчике события TimerActionEvent.
