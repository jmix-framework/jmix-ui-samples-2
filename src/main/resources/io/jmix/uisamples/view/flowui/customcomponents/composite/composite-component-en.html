Composite components are created by combining existing components together. They encapsulate UI layout and behaviour and allow you to reuse identical parts of UI in different views. Composite components can contain data-aware components and load data themselves or rely on data containers provided by the host view.

In this example, we demonstrate a composite component for editing addresses called AddressComponent. The component consists of several TypedTextField and EntityComboBox components representing the address parts. The component's constructor accepts a few infrastructure beans and creates inner components and data containers. The initContent() method is invoked when the composite component is attached to the components tree. It adds its inner components to the parent component which is defined by the type parameter (Composite<FormLayout>). The composite component defines its own API method setDataContainer() which must be invoked after the component is created in a view. The method binds the inner components to the provided data container.

After creating the composite component class, it can be used in views programmatically. See the initAddress() method of the CompositeComponentSample class.

To be able to use the new component in views XML declaratively, a few more steps are needed:

  1. Create an XSD for the component (see composite-component.xsd).
  2. Implement the component loader (see AddressComponentLoader.java).
  3. Register the component and its loader in the application (see CompositeComponentConfiguration.java).

Once these steps are completed, the composite component can be used in view XML descriptors. See the composite-component.xml file.
