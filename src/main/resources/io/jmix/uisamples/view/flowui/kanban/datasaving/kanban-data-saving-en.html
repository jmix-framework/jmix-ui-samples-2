There are two ways to save changes for the Kanban board component: manual (default) and automatic.

Using the component, it is possible to change three attributes of the entity:

  • status, swimlane - by dragging Kanban task cards into different columns
  • priority - using the corresponding drop-down list next to the text on the Kanban task card

Changes are automatically commited in the data container, but by default do not initiate saving to the database.

In manual mode, you must define SaveDelegate, which will be called each time you change cards using the Kanban board.

In automatic mode, you must set the attribute autoSave in XML to true. This will automatically set the SaveDelegate in the component, which will save the DataContext of the current view.
Be careful, the entire view data context will be saved.
