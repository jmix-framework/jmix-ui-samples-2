В данном примере демонстрируется подхода для отображения PDF-файлов из локального хранилища с использованием компонента IFrame и REST API.

Этот способ удобен, если документ хранится в сущности, в качестве FileRef.
Если в Вашем приложении настроен Generic REST API, то вы можете просто указать адрес запроса в качестве источника для IFrame (см. Files API :: Документация Jmix).
Таким образом по этом запросу придёт файл, который будет отображен стандартными средствами браузера в IFrame.

В данном примере настроен собственный контроллер запросов. См. исходный код в CustomFileDownloadController.
Чтобы такого вида запросы правильно обрабатывались приложением, необходимо добавить специальный фильтр.
См. исходный код в CustomRestSecurityConfiguration.

В исходном коде указана директива, которая разрешает использовать ресурсы в том случае, если запрос идёт с demo.jmix.io.
Более подробная информация доступна в документации к директиве: CSP: frame-ancestors - HTTP | MDN 6.
