Для создания компонента Fragment, необходимо создать класс, который будет расширять io.jmix.flowui.fragment.Fragment. Обратите внимание, что Fragment типизирован компонентом, который будет использован в качестве корневого в вашем компоненте. Также, именно этот компонент должен быть указан в качестве контента в разметке XML. В данном примере - это компонент JmixDetails.
Для привсзки XML-файла к компоненту фрагмента используется аннотация @FragmentDescriptor (см. FragmentSimpleSample.java).

Для определения компонента Fragment в экране используется тэг fragment. Вам необходимо указать полное имя класса для вашего фрагмента (см. fragment-simple.xml).

Также, как и для экранов, фрагменты имеют возможность декларативной установки подписок и установок для компонентов.
Для этого Вы можете использовать аннотации @Subscribe, @Install и @Supply.

Обратите внимание, что для компонентов внутри фрагментов не назначаются действительные ID. Для них устанавливаются скрытые ID. Для работы с ними используется FragmentUtils. Это сделано для того, чтобы избежать конфликт ID компонентов в хост-экране и фрагменте.
