Timer is a facet (non-visual component) designed to run some code at specified time intervals.

The timer works in a thread that handles user interface events, therefore it can update view components.

Timer stops working when a view it was created for is closed.

Timers are described using the timer element. It has the following attributes:

  • delay - a required attribute;
    it defines timer interval in milliseconds;
  • autostart - an optional attribute;
    when it is set to true, timer starts immediately after the view is opened. By default, the value is false, witch means that the timer will start only when its start() method is invoked;
  • repeating - an optional attribute, turns on repeated executions of the timer. If the attribute is set to true, the timer runs in cycles at equals intervals defined in the delay attribute. Otherwise, the timer runs only once - delay milliseconds after the timer start.

To execute some code on timer, subscribe to its TimerActionEvent in the view controller.
