To create your own custom component or integrate a third party JavaScript library, you need to describe its server-side and client-side parts.

Taking Slider | jQuery UI as an example, this sample shows how it can be integrated.

JS libraries and other sources can be imported from the server-side using the following annotations:

  • @Tag - defines the name of the HTML element.
  • @NpmPackage - used to define NPM dependency.
  • @CssImport - used to import corresponding styles.
  • @JsModule - used to import custom JavaScript.

Your component may require in-project front-end files, such as additional JavaScript modules. In which case, add them to the src/main/resources/META-INF/frontend directory so that they are packaged in the component JAR.

In this example, on the client-side, the ready() method can be used to initialize the corresponding JS library.

To completely integrate the component, it is necessary to support it in XML.

Below are a few steps you need to take to do it:

  1. Create an XSD Schema for the component (see custom-component.xsd).
  2. Implement the component loader that will parse it from XML into a Java-object (see SliderLoader.java).
  3. Register the component and its loader in the application (see CustomComponentConfiguration.java).
  4. Add the custom component XSD Scheme to the XML view descriptor so that you can use the component (see custom-component.xml).
