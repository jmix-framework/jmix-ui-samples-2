Фасет `Settings` предназначен для сохранения состояние компонентов пользовательского интерфейса в базе данных в таблице
`FLOWUI_USER_SETTINGS`.
Для сохранения используется сущность JPA `UserSettingsItem`.

Чтобы включить **settings** в фасете, он должен содержать `SettingsFacet`. Проще всего добавить его через XML.

Фасет с настройкой `auto="true"` пытается найти все компоненты, у которых определен ID и имеется поддержка **settings**.
Все эти компоненты сохранят и применят свои настройки.

Поддерживаемые компоненты:

- `DataGrid`: сохраняет порядок, сортировку и видимость столбцов
- `Details`: сохраняет состояние свойства <b>opened</b>
- `SimplePagination`: сохраняет свойство `maxResults`, если атрибут `itemsPerPageVisible` установлен в значение `true`
- `GenericFilter`: сохраняет состояние свойства <b>opened</b>

В данном пример для демонстрации работы фасета **settings** используются компоненты `DataGrid` и `Details`.
Изначальное состояние компонента `Details` установлено как `opened`.
Вы можете изменить это состояние, затем перейти к другому экраны и вернуться.
Последнее состояние будет восстановлено. То же самое применимо к компоненту `DataGrid`.
