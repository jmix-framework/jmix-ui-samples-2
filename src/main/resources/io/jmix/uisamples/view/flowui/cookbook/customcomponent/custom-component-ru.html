Для создания собственного компонента, или интеграции third-party JavaScript необходимо описать серверную и клиентскую часть компонента.

Для реализации будет взят Slider | jQuery UI. В данном примере показано, как интегрировать такой компонент.

JS библиотеки и другие источники могут быть импортированы с серверной части при помощи следующих аннотаций:

  • @Tag - определяет название элемента HTML.
  • @NpmPackage - используется для определения NPM зависимостей.
  • @CssImport - используется для подключения соответствующих стилей.
  • @JsModule - используется для импорта пользовательских JavaScript файлов.

Вашему компоненту могут потребоваться внутренние файлы клиентской части, например дополнительные модули JavaScript. В таком случае, добавьте их в каталог src/main/resources/META-INF/frontend, чтобы они были упакованы в компонент JAR.

В данном примере, в клиентской части используется метод ready() для инициализации необходимой библиотеки JS.

Для полной интеграции компонента необходимо обеспечить его поддержку в XML.

Ниже представлены несколько шагов, которые позволят это сделать:

  1. Создать XSD схему для компонента (с.м. custom-component.xsd).
  2. Реализовать загрузчик компонента, который будет анализировать XML и создавать по нему соответствующий Java-объект (с.м. SliderLoader.java).
  3. Зарегистрировать компонент и его загрузчик в приложении (с.м. CustomComponentConfiguration.java).
  4. Добавить собственную XSD схему в XML-файл экрана для того, чтобы использовать компонент (с.м. custom-component.xml).
